Network datasets need provenance, repeatability, and confidence scoring.

ipanalytics focuses on practical IP intelligence systems: collecting public network metadata, normalizing inconsistent provider data, joining ASN and CIDR context, validating GeoIP/MMDB outputs, and publishing compact datasets for security, abuse, fraud, routing, crawler visibility, and infrastructure analytics workflows.

IP and ASN enrichment

Projects are built around CIDR ranges, ASN ownership, hosting and CDN context, RIR and WHOIS metadata, geofeeds, source provenance, and confidence-aware enrichment.

Infrastructure visibility

Research covers VPN, proxy, Tor, hosting, crawler, AI fetcher, cloud, CDN, scanner, and reputation signals without depending on a single opaque feed.

Operational datasets

Outputs are designed for dashboards, CSV/JSON pipelines, MMDB processing, reproducible builds, quality checks, static APIs, and downstream security workflows.

Research focus

Signals that improve network context

  • VPN, proxy, Tor, hosting, CDN, cloud, crawler, scanner, and reputation signal analysis.
  • GeoIP/MMDB processing, validation, diffing, and multi-source location consensus.

Data formats and stack

Built for practical pipelines

  • Languages and tools: Go, Python, JavaScript, HTML, Linux, Docker, GitHub Actions.
  • Formats: CSV, JSON, JSONL, Parquet, MMDB, CIDR lists, static APIs, and lookup indexes.
  • Network data: ASN, BGP, RIR delegated stats, WHOIS, GeoIP, geofeeds, and provider metadata.
  • Signals: VPN, proxy, Tor, crawler, cloud, CDN, hosting, scanner, abuse, and reputation.

Why this work matters

Transparent network intelligence is easier to audit.

Many IP intelligence feeds are useful but difficult to inspect. The ipanalytics projects prioritize clear inputs, compact outputs, source-aware scoring, reproducible update paths, and operational formats that are easy to review, compare, and integrate.
